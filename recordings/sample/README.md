# Sample recording (hand-built)

`mainnet/18209181/scores.ndjson` is a small, **hand-built synthetic** recording of France (Participant1) vs Morocco (Participant2) — a stand-in for a real captured quarterfinal so CI replay tests and the replayer have deterministic data until a live recording lands. It follows the README §7.3 Scores schema exactly (each line is a `{recvMs, recvIso, id, event, data}` envelope as written by `services/ingest/src/record.ts`; `data` is a verbatim Scores JSON string). The story: 0-0 kickoff → France goal ~23' → 1-0 at half time → Morocco equalizer ~67' → 1-1 at full time (`statusSoccerId: 5`), with corners and one Morocco yellow kept monotonic and consistent between `scoreSoccer` and the `stats` map, plus one `heartbeat` frame. Replay it with `npm run replay -- --fixture 18209181 --network mainnet --speed 60` (the CLI auto-falls back to this sample when no live recording exists at `recordings/mainnet/18209181/scores.ndjson`). Once a real match is captured via `npm run record`, that recording takes precedence and this sample stays only as a CI/demo fixture.
